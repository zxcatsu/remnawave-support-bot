## ✨ Возможности

| | |
|---|---|
| 🔌 **Интеграция с БД RemnaWave** | При открытии тикета бот подтягивает статус подписки, дату окончания и остаток трафика прямо из Postgres панели. |
| 💬 **Форумная система (Topics)** | Каждое обращение — отдельная тема в админской группе с карточкой клиента и inline-кнопками управления. |
| ⭐ **Отзывы с модерацией** | Пользователь оставляет отзыв → админ одобряет/отклоняет → публикация в канал. Отзывы хранятся в БД и переживают рестарт. |
| ⏰ **Автозакрытие тикетов** | Неактивные тикеты закрываются автоматически через `AUTO_CLOSE_HOURS` часов. |
| 🚫 **Баны** | Блокировка спамеров в один клик прямо из карточки тикета. |
| 📝 **Логирование** | Одновременно в `docker compose logs` и в файл `bot.log` с ротацией (5 × 5 МБ). |
| 🧙 **Мастер установки** | Интерактивный `setup.sh` — настройка и запуск без ручного редактирования конфигов. |

## 🚀 Быстрая установка

Интерактивный мастер сам проверит Docker, соберёт `.env` и запустит бота:

```bash
git clone https://github.com/zxcatsu/remnawave-support-bot.git
cd remnawave-support-bot
chmod +x setup.sh
./setup.sh
```

### Что спросит установщик
1. **Токен бота** — получите у [@BotFather](https://t.me/BotFather).
2. **ID админ-группы** — начинается с `-100` (узнать можно через [@RawDataBot](https://t.me/RawDataBot)).
   > ⚠️ В настройках группы включите тумблер **«Темы» (Topics)** и выдайте боту права администратора.
3. **ID темы логов/банов** — создайте тему, отправьте туда сообщение, скопируйте ссылку `https://t.me/c/123.../42` — число в конце (`42`) и есть ID.
4. **Данные БД RemnaWave** — пароль от Postgres и имя Docker-сети панели (`docker network ls`).
5. **Отзывы** *(опционально)* — тема модерации и канал для публикации.

## ⚙️ Конфигурация (`.env`)

Скопируйте `.env.example` в `.env` и заполните (или используйте `setup.sh`):

| Переменная | Описание | По умолчанию |
|---|---|---|
| `PROJECT_NAME` | Название проекта в сообщениях бота | `My VPN Support` |
| `TELEGRAM_TOKEN` | Токен от @BotFather | — |
| `ADMIN_GROUP_ID` | ID группы с топиками (`-100...`) | — |
| `BANS_TOPIC_ID` | ID темы для логов и банов | `1` |
| `PG_HOST` / `PG_DB` / `PG_USER` / `PG_PASS` | Доступ к Postgres панели | — |
| `EXTERNAL_NETWORK_NAME` | Docker-сеть панели | `remnawave-network` |
| `AUTO_CLOSE_HOURS` | Часов без активности до автозакрытия (`0` = выкл) | `24` |
| `REVIEWS_TOPIC_ID` | ID темы модерации отзывов | `1` |
| `REVIEWS_CHANNEL` | Канал для публикации отзывов | `@my_reviews_channel` |
| `TZ` | Часовой пояс | `Europe/Moscow` |

## 💡 Использование

1. Пользователь пишет боту и нажимает **«🎫 Открыть новый тикет»**.
2. В админской группе создаётся тема с ID тикета и карточкой клиента из БД панели.
3. Администраторы отвечают прямо в теме — бот пересылает ответы пользователю.
4. Кнопки под карточкой: **🔒 Закрыть**, **🚫 Забанить** / **✅ Разбанить**.
5. Неактивные тикеты закрываются сами через `AUTO_CLOSE_HOURS`.

## ❓ FAQ

**Бот не показывает данные подписки («Ошибка связи с БД»)?**
Проверьте, что контейнер бота и Postgres панели в одной Docker-сети (`EXTERNAL_NETWORK_NAME`) и что имена таблиц/полей совпадают с вашей версией RemnaWave.

**«Отзыв не найден» при модерации?**
Больше не воспроизводится: отзывы хранятся в SQLite и переживают рестарт контейнера.

**Как отключить автозакрытие тикетов?**
Установите `AUTO_CLOSE_HOURS=0`.
